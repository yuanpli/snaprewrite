1) SnapRewrite 具体怎么触发改写？
支持三种触发方式：拖选文本后触发、双击词语触发、以及快捷键触发。你可以在设置里单独开启或关闭每一种方式。触发后会弹出操作面板，你可以选择“改写”或“自定义命令”。
2) 为什么必须申请“辅助功能（Accessibility）”权限？
因为应用需要在其他 App 内识别“当前选中的文本”并实现“原位替换”，这属于系统受控能力。没有该权限时，SnapRewrite 无法感知外部应用中的选区，也就无法完成主流程。
3) 会读取我所有内容吗？会上传本地文档吗？
不会。SnapRewrite 只在你主动触发时处理“当前选中的文本片段”。不会自动扫描整篇文档，也不会后台批量上传文件。配置与偏好设置主要保存在本地设备中。
4) 文本会被发送到哪里？
文本会发送到你在应用中选择的 AI 服务商接口（如 OpenAI 兼容接口、智谱等）进行改写处理。不同服务商的数据处理和保留策略不同，请同时查阅对应服务商条款。
5) API Key 存储在哪里，安全吗？
用户填写的 API Key 存储在 macOS Keychain（钥匙串）中，而不是明文写在普通配置文件中。你可随时在设置中替换或清空密钥。若设备本身遭到入侵，任何本地数据都有风险，建议配合系统级安全策略使用。
6) 免费版和 Pro 的区别是什么？
免费版有每日额度限制；Pro 解锁后可使用更完整能力（如更多模型/高级自定义能力，具体以当前版本功能为准）。Pro 通过 Apple 内购解锁，权益与 Apple ID 绑定。
7) 我换了 Mac，如何恢复 Pro？
在应用中使用“恢复购买”即可通过 Apple StoreKit 同步当前 Apple ID 的购买状态。请确认登录的是购买时使用的同一 Apple ID。
8) 为什么有时会读取剪贴板？
在部分非标准文本控件中，系统选区接口不稳定。SnapRewrite 会在触发场景下短暂探测复制结果来识别选中内容，并尽量恢复原始剪贴板文本，以减少对用户工作流影响。
