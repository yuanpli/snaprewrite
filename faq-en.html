SnapRewrite FAQ (English)

This page clarifies permissions, privacy behavior, provider setup, billing flow, and legal links for users and App Store reviewers.

1) How do I trigger rewriting in SnapRewrite?
SnapRewrite supports three triggers: drag selection, double-click selection, and custom hotkey trigger. You can enable or disable each trigger in Settings based on your writing workflow.
2) Why is Accessibility permission required?
Accessibility permission is required by macOS so the app can detect selected text in other apps and provide in-place replacement. Without this permission, the main cross-app rewriting flow cannot run.
3) Does SnapRewrite read all my documents?
No. SnapRewrite only processes the text you actively select and explicitly trigger. It does not automatically scan full documents or batch-upload local files in the background.
4) Where is selected text sent?
Selected text is sent to the AI provider you configured (for example, OpenAI-compatible endpoints or Zhipu) to generate rewritten output. Data handling policies are defined by each provider you choose.
5) Where are API keys stored?
User-provided API keys are stored in local macOS Keychain rather than plain text app preferences. You can rotate or clear keys in Settings at any time.
6) What is the difference between Free and Pro?
Free includes a daily quota limit. Pro unlocks a broader feature set (such as expanded model/custom capabilities, depending on the current version) and is activated via Apple in-app purchase.
7) How do I restore Pro on a new Mac?
Use the in-app Restore Purchases action. Apple StoreKit will sync entitlement for the Apple ID used during the original purchase.
8) Why can clipboard probing happen in some apps?
Some non-standard text controls do not reliably expose selection APIs. In those cases, SnapRewrite may briefly probe copy results in trigger context and then restore clipboard text to minimize workflow disruption.
